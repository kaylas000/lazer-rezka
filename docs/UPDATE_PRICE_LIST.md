# Как обновить прайс в PDF (price-list.pdf)

Здесь простые шаги без лишних терминов. Файл с прайсом на сайте лежит здесь: **`assets/files/price-list.pdf`**.

## Откуда берутся телефон и почта в PDF

Скрипт читает файл **`_data/business.yml`**. Там указаны:

- телефон (`phone`)
- почта (`email`)
- режим работы (`hours` и при необходимости блок `openingHours`)
- адрес (`address`)

Если поменяли телефон или почту на сайте — сначала обновите **`_data/business.yml`**, потом заново соберите PDF (см. ниже).

## Что поставить один раз на компьютере

Нужны **Python 3** и шрифты **DejaVu** (для русских букв в PDF). На Linux обычно уже есть пакет вроде `fonts-dejavu-core`.

Зависимости Python для скрипта:

```bash
cd /путь/к/репозиторию/lazer-rezka
pip install -r requirements/price-list.txt
```

(Можно через `pip install --user`, если так принято у вас.)

## Какую команду запустить

Из **корня репозитория** (там же, где лежат папки `assets`, `_data`, `scripts`):

```bash
python3 scripts/build_price_list.py
```

В конце должна появиться строка:

`OK: wrote assets/files/price-list.pdf`

## Как проверить, что в PDF правильные контакты

После генерации:

```bash
strings assets/files/price-list.pdf | grep -E 'tel:|mailto:'
```

Для текущего телефона **`+7 (985) 456-37-64`** в ссылке будет вид **`tel:+79854563764`** (цифры без пробелов и скобок).  
Для почты **`info@lasercut.ru`** будет **`mailto:info@lasercut.ru`**.

Убедитесь, что **нет** старых или чужих адресов, если вы их меняли.

## Что менять, если нужны другие цены в таблице

Сейчас таблицы в PDF **зашиты в скрипте** `scripts/build_price_list.py` и совпадают с блоком прайса на странице калькулятора (**`pages/calculator.html`**).

- Чтобы изменить **цифры или строки таблиц** — отредактируйте списки `laser_rows` и `extra_rows` в **`scripts/build_price_list.py`** и при необходимости тот же текст на сайте в **`pages/calculator.html`**, чтобы сайт и PDF не расходились.

## Что сделать после проверки

1. Откройте сгенерированный **`assets/files/price-list.pdf`** глазами (откроется в любом просмотрщике PDF).
2. Закоммитьте только изменение PDF (и при необходимости правки в `business.yml` или скрипте) — отдельным коммитом в своей ветке, как обычно для сайта.

Автоматическая сборка PDF в GitHub Actions в этом репозитории **не настроена** намеренно: генерация только вручную, чтобы не трогать деплой и не тратить лишние минуты CI без нужды.
